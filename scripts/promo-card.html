wolfpack-logsync

Auto-upload your EQ parse. No copy-paste. No clipboard. Just kill the boss.
✓ Zero dependencies ⚡ Node.js only 🔒 Privacy first 🗂 Single file
⚙️  How It Works
  • 📄 Tails your EQ log from the end of file forward — never reads your full multi-GB log
  • 🔍 Detects combat events as they happen — damage, heals, deaths, casts
  • 🚫 Strips private channels locally at the byte level, before any parsing
  • 📤 Uploads compact JSON to the bot at the end of each fight — nothing else
  • 🤖 Bot posts the parse card automatically — DPS, healers, deaths, gaps
✨  What You Unlock
  • 📊
    Live parse cards DPS rankings auto-post on boss death — no /parse needed
  • 🩺
    Healer leaderboard Total healed · ticks · targets · CH chain visibility
  • ⚠️
    Heal gap warnings Detects stretches >8s without a heal on the main tank
  • 💀
    Death scoreboard Per-fight deaths + ⚔️ flags Paladin/SK riposte deaths
  • 🎯
    Raid night tagging Sun/Wed/Thu parses tagged for /parsestats filtering
🔒  Privacy Guarantee
✕ Officer chat dropped first
✕ Tells dropped first
✕ Guild chat dropped first
✕ Raid say dropped first
✕ Group chat dropped first
✓ Combat only what's sent
Private channels are filtered at the byte level before any parsing or buffering. Nothing from those channels is ever parsed, stored, or uploaded. Only damage, heal, and death events leave your machine.

🤝
More parsers = more complete data. When multiple guild members run wolfpack-logsync on the same boss fight, the bot automatically merges all perspectives into one card. Each parser sees different heal targets, different damage events — combined they give a much more accurate picture of the raid.
▶  One Command to Run
Requires Node.js 18+ (already on most machines). Runs in a terminal alongside EQ.
$ node wolfpack-logsync --log "C:\EverQuest\Logs\eqlog_YourName_pq.proj.txt" --watch
